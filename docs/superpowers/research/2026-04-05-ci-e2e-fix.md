# CI E2E Test Failure: Research Report

**Date:** 2026-04-05
**Status:** Research complete, ready for implementation

---

## Problem

32 of 84 Playwright E2E tests fail in CI. GitHub Actions lacks Supabase credentials.

## Root Cause

`getServerClient()` in `src/lib/supabase.ts` **throws** when env vars are missing. Pages that wrap calls in try/catch degrade gracefully; pages that don't, crash.

Most pages DO have defensive coding (try/catch), so 32 failures may stem from the build step or dev server startup failing entirely.

---

## Test Inventory (84 total = 42 specs x 2 viewports)

| File | Specs | Needs Supabase? |
|------|-------|----------------|
| pages.spec.ts | 21 | Mostly no. Demo brief uses mock data. Upload page has `safeGetDb()` fallback. |
| showcase.spec.ts | 12 | Pages have try/catch fallbacks. Should degrade gracefully. |
| feedback.spec.ts | 4 | No. All use demo mock data. |

**Key finding:** Most pages have defensive coding. The 32 failures likely stem from build/dev-server issues, not page rendering.

---

## Options

### Option A: GitHub Secrets (IMMEDIATE FIX)
- Add `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY` in GitHub Settings > Secrets
- CI config already references them -- zero code changes
- **Risk:** Service role key in CI, tests depend on production DB state

### Option B: Test Tagging (skip DB tests in CI)
- Use `test.skip()` when env vars missing
- Loses 38% coverage -- not recommended

### Option C: MSW Mocking
- Mock Supabase PostgREST HTTP calls with fixture JSON
- High setup cost (2-3 days), MSW + Next.js SSR is complex
- Ongoing maintenance burden
- Not recommended for this project size

### Option D: Supabase Local in CI (MEDIUM-TERM)
- `supabase start` in GitHub Actions (Docker pre-installed)
- Run migrations + seed data against local instance
- Deterministic, credential-free, tests migrations too
- Setup: 1-2 days, needs `config.toml` + CI seed script
- **Best for open-source project with external contributors**

---

## Recommendation

1. **Today:** Option A -- add GitHub secrets (5 min, gets CI green)
2. **Investigate:** Verify actual failure mode (dev server crash vs page render vs assertion mismatch)
3. **v1.1 timeframe:** Option D -- Supabase local in CI for contributor-friendly testing
